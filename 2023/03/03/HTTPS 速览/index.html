HTTPS 速览

2023-03-03

HTTPS 到目前为止没有任何漏洞，有问题也是客户端的问题。

回顾

为解决 HTTP/1.1 的不安全，提出了 HTTPS：

不安全 HTTPS 解决策略
明文传输不安全：内容不加密，会被窃听 信息加密，公钥 + 私钥
不去验证通信方的身份：第三方骗子冒充服务端 摘要算法 + 数字签名
无法验证报文完整性：响应资源的内容可能遭到篡改 CA 颁发数字证书：解决公钥传输问题

三种加密方式

  • 单向加密：只能加密，不能解密；输入一样，输出一样，输入不同，输出不同
  • 对称加密：加密和解密都用同一把密钥；速度快
  • 非对称加密：公钥加密，私钥解密，或者私钥加密，公钥解密；速度非常慢
    • 公钥加密，私钥解密：用来加密内容
    • 私钥加密，公钥解密：用来验证身份

HTTPS 实现安全传输的策略

参考资料

  • 混合加密（保护会话密钥）
  • 摘要算法（原数据太大不方便加密，减小加密数据的大小） + 数字签名（防篡改）
  • 数字证书（保证公钥安全）

😅 HTTPS 建立连接过程

TLS 握手

  • 基于 RSA 算法
  • 基于 ECDHE 算法

HTTP 与 HTTPS 的区别

  • HTTP 明文传输；HTTPS 在传输层和应用层之间加入 SSL/TLS 协议，实现报文加密传输
  • HTTP 只需三次握手；HTTPS 除了需要三次握手，还需要四次 TLS 握手
  • HTTP 默认端口号为 80；HTTPS 默认端口号为 443
  • URL 上 HTTP 是 “http://”；HTTPS 是 “https://”

HTTPS 的优化

  • 硬件优化：用更贵的 CPU
  • 软件优化：升级 Linux、OpenSSL
  • 协议优化：选用 ECDHE 密钥交换算法替换 RSA 算法；把 TLS 1.2 升级成 TLS 1.3
  • 证书优化：选择 ECDSA 证书，而不是 RSA 证书
  • 会话复用：Session ID —> Session Ticket —> Pre-shared Key
