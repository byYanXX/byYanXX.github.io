美团

一面 2023.04.21（1h10min）

项目

  1. 介绍一下项目里的核心模块，有哪几个核心数据表，它们怎么设计的？
  2. Redis 缓存主要哪些地方在用？
  3. 热点数据是事先定义好的吗？（答：是）追问，新增来的商品也有可能成为热点商品，你如何处理？（答：比如 500 个商品，一定周期更换后面的 100 个商品）
  4. 详细讲一下分布式锁解决缓存击穿这里是怎么做的，如何实现的？
  5. 为了不让击穿，使用分布式锁，反而降低了性能，是不是不太合理，有什么更好的方案？（脑子抽了没答上来，这种数据就不用设置过期时间）
  6. 具体什么功能会用到 ThreadLocal？讲一下使用流程？

八股

  1. TCP 和 UDP 区别？
  2. TCP 如何保证可靠性？
  3. 输入 URL 后会做哪些事情？
  4. == 和 equal 区别，两个 User 类用 == 和用 equal 区别？
  5. Integer a = 1; Integer b = 1; 判断 a == b？
  6. Integer a = 1000; Integer b = 1000; 判断 a == b？（脑子抽了，竟然说的 true）
  7. String a = “abc”; String b = “abc”; 判断 a == b？
  8. ThreadLocal 底层如何设计的？使用场景？
  9. volatile？
  10. 手写一下双重检验的单例？这里 volatile 有什么作用？不加 volatile 会出现什么现象？为什么要两次判断是否为空？

算法

华为

一面 2023.05.25（1h）

八股（15 min）

  1. 你是怎么看待数据结构的？简单讲讲你了解的数据结构（刚讲完链表、数组、栈，就打断了）
  2. 你能说说哈希表的原理吗？解决哈希冲突的算法？
  3. 描述一下 IP 地址和端口的作用？
  4. 有些过软件通信方面的服务吗？你的第二个项目有用到相位机的经验吗？
  5. 讲讲数据库 2NF 和 3NF 的差异？
  6. 你有自己设计数据库吗？怎么设计订单模块的数据库的？
  7. 数据量有多大？（回答 200~500w）
  8. 你怎么对 MySQL 进行调优的？（200~500 w 数据量足够 MySQL 支持，我没有涉及调优）
  9. 那 MySQL 数据量达到多少时会遇到瓶颈？有没有做过压测？（2000w 左右）
  10. 如果数据到达 2000w 你有做哪些预案？

项目（10min）

  1. 你做的部分用了哪些技术，遇到了哪些问题？
  2. 解决重复提交为什么不用 db 的事务？
  3. Redis 挂掉的话如何处理？
  4. Redis 如何部署的？是三主三备的吗？
  5. RabbitMQ 的延迟队列机制你了解吗？它是落盘存储的还是内存存储的？可靠性怎么样？
  6. RabbitMQ 崩了的话，如何解锁库存呢？（我没深入了解）

算法（要求 20min 以内）

合并两个无序链表

我的方法：先存入数组，排序，再合并两个有序链表（力扣 21）（花了 18 min）

面试官：有没有更优的解法，能不能直接合并，不提前对两个链表进行排序，不需要写，说出思路就好

我：可以用归并的方法（花了 2min）

后记

总的来说，八股部分没有自己想象中问的那么深，但「基础一定要扎实」，在比较偏、难、怪的问题不要浪费太多时间。

另外，华为的八股没有那么围绕 Java，更多的是计算机基础方面的，备战秋招时要注意一下。

在项目上，要有自己的思考，为什么选择 A 方法而不选择 B 方法、A 方法为什么这么用、……多问问自己，多思考背景。

简历里的东西一定要非常熟悉！写了的知识点被问住了的话，扣大分 :(

反问环节少问烂大街的那些问题，比如：培养机制、时间安排、……，一定要问一些有价值的问题，另外可以多问几个，别只问俩。

最后，焦虑没有一点用，都是围城，没有消息时担心 0 offer，有了之后担心学不到东西、担心没法转正、担心薪资低、……尽自己最大努力吧，只求过程，不问结果，过分追求结果，到最后真就没结果。
