两种数据类型
---
1. 基本类型：
  - 整型：byte (1个字节)、short (2个字节)、int (4个字节)、long (8个字节)，一个字节就是 1byte，1024 字节就是 1K
  - 浮点型：float (4个字节)、double (8个字节)
  - 布尔型：boolean (理论上存储布尔类型只需要 1 bit，但是通常 JVM 内部会把 boolean 表示为 4 字节整数)
  - 字符型：char (支持ASCII、也支持一个Unicode字符)
2. 引用类型：
  - String 等

- byte：-128 ~ 127
- short: -32768 ~ 32767
- int: -2147483648 ~ 2147483647
- long: -9223372036854775808 ~ 9223372036854775807

整数运算
===

移位运算
---
**针对整数，浮点型不能做位运算或移位运算**

- 左移
```
int n = 7;       // 00000000 00000000 00000000 00000111 = 7
int a = n << 1;  // 00000000 00000000 00000000 00001110 = 14
int b = n << 2;  // 00000000 00000000 00000000 00011100 = 28
int c = n << 28; // 01110000 00000000 00000000 00000000 = 1879048192
int d = n << 29; // 11100000 00000000 00000000 00000000 = -536870912
// 左移29位时，由于最高位变成1，因此结果变成了负数。
```
- 右移
最高位 1 不动，结果仍是负数：
```
int n = -536870912;
int a = n >> 1;  // 11110000 00000000 00000000 00000000 = -268435456
int b = n >> 2;  // 11111000 00000000 00000000 00000000 = -134217728
int c = n >> 28; // 11111111 11111111 11111111 11111110 = -2
int d = n >> 29; // 11111111 11111111 11111111 11111111 = -1
```
- 无符号的右移
最高位 1 跟着动，负数>>>右移会变为正数：
```
int n = -536870912;
int a = n >>> 1;  // 01110000 00000000 00000000 00000000 = 1879048192
int b = n >>> 2;  // 00111000 00000000 00000000 00000000 = 939524096
int c = n >>> 29; // 00000000 00000000 00000000 00000111 = 7
int d = n >>> 31; // 00000000 00000000 00000000 00000001 = 1
```

**注意：**
- byte 和 short 类型在移位时会先转为 int 类型再位移。
- 在运算过程中，如果参与运算的两个数类型不一致，那么计算结果为较大类型的整型。(超出范围的强制转型会得到错误的结果)

位运算
---
与 &
```
n = 0 & 0; // 0
n = 0 & 1; // 0
n = 1 & 0; // 0
n = 1 & 1; // 1
```
或 |
```
n = 0 | 0; // 0
n = 0 | 1; // 1
n = 1 | 0; // 1
n = 1 | 1; // 1
```
非 ~
```
n = ~0; // 1
n = ~1; // 0
```
异或 ^
```
n = 0 ^ 0; // 0
n = 0 ^ 1; // 1
n = 1 ^ 0; // 1
n = 1 ^ 1; // 0
```

浮点数运算
===

**浮点数有个非常重要的特点，就是浮点数常常无法精确表示。**

浮点数 0.1 在计算机中就无法精确表示，因为十进制的 0.1 换算成二进制是一个无限循环小数，很显然，无论使用 float 是 double，都只能存储一个 0.1 的近似值。但是，0.5 这个浮点数又可以精确地表示。
