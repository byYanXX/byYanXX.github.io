2022-11-29

SpringMVC

什么是 SpringMVC

MVC 是模型（Model）、视图（View）、控制器（Controller）的缩写，旨在将业务「逻辑、视图显示、数据」分离来组织代码。

MVC 是众多设计模式中的一种，SpringMVC 是一款优秀的 MVC 框架。

一般将业务分层：

  • Service：业务处理
  • Dao：数据库操作
  • Entity：实体类
  • Controller：控制层，将数据传递给前端

SpringMVC 核心组件

  • DispatcherServlet：核心的中央处理器，负责请求、分发、响应客户端
  • HandlerMapping：处理器映射器，根据 uri 去匹配查找能处理的 Handler，将请求涉及到的拦截器和 Handler 一起封装
  • HandlerAdapter：处理器适配器，根据 HandlerMapping 找到 Handler，适配执行对应的 Handler
  • Handler：请求处理器，处理实际请求的处理器
  • ViewResolver：视图解析器，根据 Handler 返回的视图/逻辑视图，解析并渲染后，传递给 DispatcherServlet 响应客户端

SpringMVC 工作原理

  1. 客户端发出请求，DispatcherServlet 拦截请求
  2. DispatcherServlet 根据请求信息调用 HandlerMapping，HandlerMapping 根据 uri 去匹配查找能处理的 Handler，并将请求涉及到的拦截器和 Handler 一起封装
  3. DispatcherServlet 调用 HandlerAdapter 适配执行 Handler
  4. Handler 处理客户端请求，返回一个 ModelAndView 对象（包含数据对象 Model 和逻辑视图 View）给 DispatcherServlet
  5. ViewResovler 根据逻辑视图查找实际的视图
  6. DispatcherServlet 将返回的 Model 传给 View
  7. 最后将 View 返回给客户端

统一异常处理怎么实现？

使用 @ControllerAdvcie + @ExceptionHandler 这两个注解。

上述操作，会给所有或者指定的 Controller 织入异常处理的逻辑（AOP），当 Controller 中的方法抛出异常的时候，由被@ExceptionHandler 注解修饰的方法进行处理。

而异常具体被「哪个被 @ExceptionHandler 注解修饰的方法」处理异常，则是由 ExceptionHandlerMethodResolver 中 getMappedMethod 方法决定。
