Spring、SpringBoot、SpringMVC 关系

SpringMVC 是 Spring 中的一个模块，主要赋予 Spring 快速构建 MVC 架构的 Web 程序的能力。

MVC 是模型（Model）、视图（View）、控制器（Controller）的缩写，旨在将业务「逻辑、视图显示、数据」分离来组织代码。

Spring 在配置时需要对 xml 文件或 Java 文件进行显示配置，比较麻烦，由此诞生 SpringBoot。

IOC

什么是 IOC

IOC 就是控制反转，不是技术，而是一种设计思想。在创建对象及其依赖关系时，以前由程序员显式写出相关代码，而 IOC 思想则是将这些过程交给 Spring 来处理。

  • 控制：对象创建的权力
  • 反转：获得依赖对象的方式交给 Spring IOC 容器来完成

IOC：

对象之间的依赖关系由 IOC 管理，并由 IOC 容器完成对象的注入。有了 IOC，我们想要创建一个对象，无需再去 new，而是通过简单的配置或注解，即可拿到创建好的对象。

IOC 解决了什么问题？

  1. 对象之间耦合度（依赖程度）降低
  2. 资源便于管理，通过 Spring 容器可以很容易实现一个单例

IOC？DI？

IOC 容器是 Spring 实现 IOC 的载体，IOC 容器实际上就是个 Map，里面存各种对象。

DI（依赖注入）则是 IOC 最常见也最合理的一种实现方式。

什么是 Bean

Bean 指的就是被 IOC 容器管理的对象。

通过 xml 配置文件、Java 配置类或注解的方式来告诉 IOC 容器需要管理哪些对象。

将一个类声明为 Bean 的注解有哪些？

  • @Component：通用注解。可以标注任意类为 Spring 组件，不知道 Bean 属于哪一层，可以使用该注解
  • @Repository：对应 Dao 层，主要用于数据库相关操作
  • @Service：对应 Service 层，主要涉及一些复杂的逻辑，会与 Dao 层交互
  • @Controller：对应 Controller 层，主要用于接受用户请求并调用 Service 层返回数据给前端

@Component 和 @Bean 的区别？

@Component @Bean
作用于类 作用于方法
自动侦测、自动装配到 Spring 容器中 在该注解下的方法中定义产生这个 bean
自定义性更强，有些时候只能通过该注解注册 bean，比如第三方类库中的类要装配到 Spring 容器时，只能通过该注解实现

@Autowired 和 @Resource 区别？

@Autowired @Resource
提供者 Spring JDK
注入方式 默认 byType 默认 byName
参数 只有一个 required 表示是否允许对象为 null 两个参数 name 和 type
显式指定注入方式 如果有多个相同类型，byType 失效，需要结合 @Qualifier 实现 byName 注入 有两个常用的属性 name、type，可以显示指定 byName 或 byType 或 byName + byType
作用目标 构造方法、方法、参数、成员变量、注解 类、成员变量、方法

Bean 的作用域有哪些？

  • singleton：IOC 容器中只有唯一的 bean 实例。Spring 中 bean 的默认作用域是单例的
  • prototype：每次获取都会得到一个新的 bean 实例，即连续两次 getBean() 会得到两个不同的 bean
  • request：每一次 HTTP 请求都会产生一个新的 bean 实例（仅 Web 应用可用）
  • session：每一次来自新 session 的 HTTP 请求都会产生一个新的 bean（仅 Web 应用可用）
  • application/global-session：每个 Web 应用启动时创建一个 bean（仅 Web 应用可用）
  • websocket：每一次 websocket 会话都会产生一个 bean（仅 Web 应用可用）

单例 Bean 的线程安全问题？

单例 Bean 存在线程安全问题，多个线程同时操作同一个对象时会发生竞争。解决：

  • Bean 中尽量避免定义可变的成员变量
  • 类中定义一个 ThreadLocal 成员变量，将可变的成员变量存到 ThreadLocal 里

大部分时候，Bean 是无状态的（没有实例变量），这种情况下 Bean 是线程安全的。

Bean 生命周期？

  1. Spring 启动，查找并加载需要被 Spring 管理的 Bean，Bean 容器通过 Java Reflection API 对 Bean 进行实例化

  2. Bean 实例化后，如果涉及一些属性值，则通过 set() 方法设置 Bean 的属性

  3. 如果 Bean 实现了 BeanNameAware 接口，Spring 将 Bean 的 id 传递给 setBeanNamed() 方法

  4. 如果 Bean 实现了 BeanClassLoaderAware 接口，Spring 将调用 setBeanClassLoader() 方法，传入 ClassLoader 对象的实例

  5. 如果 Bean 实现了 BeanFactoryAware 接口，Spring 将调用 setBeanFactory() 方法，传入 BeanFactory 对象的实例

  6. 如果 Bean 实现了 ApplicationContextAware 接口，Spring 将调用 setApplicationContext() 方法，传入应用的上下文

  7. 如果 Bean 实现了 BeanPostProcessor 接口，Spring 将调用 postProcessBeforeInitialization() 方法

  8. 如果 Bean 实现了 InitializingBean 接口，Spring 将调用 afterPropertiesSet() 方法

    如果 Bean 使用 init-method 声明了初始化方法，该方法也会被调用

  9. 如果 Bean 实现了 BeanPostProcessor 接口，Spring 将调用 postProcessAfterInitialization() 方法

  10. 此时 Bean 已经准备就绪，可以被应用程序使用

  11. 如果 Bean 实现了 DisposableBean 接口，Spring 将调用 destroy() 方法，销毁 Bean

    如果 Bean 在配置文件中的定义包含 destroy-method 声明销毁方法，也会销毁 Bean

AOP

什么是 AOP

AOP （面向切面编程）能够将与业务无关，却为业务的模块所共同调用的逻辑或责任（事务处理、日志管理、权限控制等）封装起来。

即，通过预编译和运行期间动态代理实现程序的统一维护技术，是 OOP 的延续。旨在减少代码重复、降低模块间的耦合度、利于功能扩展。

Spring AOP 基于「动态代理」：

  • 如果要代理的对象实现了某个接口：使用「JDK Proxy」去创建代理对象
  • 代理的对象没有实现接口：无法使用「JDK Proxy」，使用的是「CGLib Proxy」生成一个被代理的对象的子类作为代理

可以使用「AspectJ AOP」。

Spring AOP 和 AspectJ AOP 区别？

  • Spring AOP 属于运行时增强，AspectJ AOP 属于编译时增强。

  • Spring AOP 基于代理（Proxying），AspectJ AOP 基于字节码操作

  • Spring AOP 已经集成了 AspectJ，切面太多时选择 AspectJ

AspectJ 定义的通知类型有哪些？

  • Before（前置通知）：目标对象的方法调用之前触发
  • After（后置通知）：目标对象的方法调用之后触发
  • AfterReturning（返回通知）：目标对象的方法调用完成，返回结果值之后触发
  • AfterThrowing（异常通知）：目标对象的方法运行中抛出异常后触发
  • Around（环绕通知）：编程式控制目标对象的方法调用

多个切面的执行顺序如何控制？

  • 使用 @Order 注解直接定义切面顺序
  • 实现 Ordered 接口重写 getOrder() 方法
